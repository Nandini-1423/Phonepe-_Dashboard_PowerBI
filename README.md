# Interactive PhonePe Transaction Analytics Dashboard
End-to-end Power BI solution built to visualize and understand payment behavior, service performance, and failure trends in digital transactions

## Executive Summary

This project presents an end-to-end Power BI analytics solution built to analyze digital payment transactions across multiple services such as Insurance, Loans, Money Transfers, and Recharge & Bills.
The goal is to convert raw transactional data into actionable business insights for operational monitoring and strategic decision-making.

## Business Problem

Digital payment platforms handle millions of transactions daily. Without proper analytics:

- Payment failure reasons remain unclear

- High-value services are difficult to identify

- Transaction health cannot be monitored effectively

This dashboard addresses these challenges by providing a centralized, interactive BI view.

## Key Insights

- Total Transactions & Total Amount analysis

- Success vs Failed Payments with root-cause breakdown

- Service-wise performance comparison

- Monthly transaction trends for growth tracking

- Identification of high-failure and high-value services
  
## Dashboard Features

- Multi-page interactive Power BI dashboard

- KPI cards for executive summary

- Dynamic date slicer across all pages

- Drill-down and cross-filtering

- Page navigation for guided storytelling

 ## Tools & Technologies

- Power BI

- DAX (Data Analysis Expressions)

- Power Query

- Data Modeling (Star Schema)

## Core Business Insights
1. High-Level Transaction Analytics

• Total Transaction Volume & Value: Comprehensive visualization of all transactions within the selected time period, enabling stakeholders to measure usage trends at a glance.

• Success vs Failed Payments: Transparent tracking of payment success rates and failure instances — crucial for improving customer experience and operational uptime.

2. Failed Payments Analysis — Key Operational Metric

Understanding failed transaction patterns is essential for any digital payment platform:

Root Causes Identified:

- Insufficient Balance

- Server Errors

- Incorrect Input (PIN/Details)

- Bank Denials

 Business Impact: High failed transaction rates can erode user trust and reduce platform adoption. 
 This dashboard accurately surfaces failure reasons, enabling prioritized action on systems, UX improvements, or failure mitigation strategies

 3. Time-Based Trend Insights

The Amount vs Month line chart displays how transaction volume and value evolve over time.
- Key Insight: Seasonality and temporal shifts in transaction behavior are visible, which helps planning promotional campaigns, resource allocation, and system capacity planning.

4. Service-Level Contribution Analysis

Each major service category (Insurance, Loans, Money Transfers, Recharge & Bills) is analyzed, illustrating:

-  Which services drive highest usage
- Which contribute greater transaction value
- How different sub-categories behave (e.g., types of insurance or loan products)

This helps cross-functional teams optimize offerings and personalize product strategies.
